## 1. Visão Geral

Esta release implementa a metodologia do **Superpowers** (obra/superpowers) no CoreOps, introduzindo:

- **BrainstormAgent** com diálogo Socrático
- **Checkpoint Manual** (artesanato) que exige confirmação explícita do usuário
- **ArchitectAgent** para especificação de arquitetura
- **SkillRegistry** com skills especializadas por framework
- **ErrorStore** para memória de erros e soluções

## 2. BrainstormAgent — Modo Superpowers

### 2.1 Fluxo Anterior (v0.1.x)

```typescript
// Execução única — retorna JSON automaticamente
const result = await agent.execute({ project, description })
```

### 2.2 Novo Fluxo (v0.2.0)

```typescript
// Modo interativo — faz perguntas até aprovação
const result = await agent.execute({ project, description })
// Se result._is_interactive === true:
//   - Pausa pipeline
//   - Espera resposta manual
//   - Continua diálogo até session_state === 'approved'
```

### 2.3 Estados da Sessão

| Estado | Descrição |
|--------|-----------|
| `initial` | Descrição inicial recebida |
| `clarifying` | Fazendo perguntas de clarification |
| `approaches` | Propondo 2-3 alternativas |
| `design` | Apresentando design em seções |
| `approved` | Design aprovado — pode avançar |

### 2.4 Campos Interativos

```typescript
interface BrainstormResult {
  // ... campos existentes
  _is_interactive: boolean
  _session_state: 'clarifying' | 'approaches' | 'design' | 'approved'
  _approaches?: string[]
  _pending_answer?: string
}
```

## 3. Checkpoint Manual (Artesanato)

### 3.1 O Problema

Em v0.1.x, checkpoints eram resolvidos automaticamente pelo LLM, sem validação humana.

### 3.2 A Solução

Checkpoint manual exige `confirm: true` explícito:

```typescript
// Usuário responde manualmente
await coreops_answer({
  answers: { "q1": "A" },
  confirm: true  // OBRIGATÓRIO para checkpoints manuais
})
```

### 3.3 Tipo checkpoint

```typescript
interface CheckpointState {
  phase: PipelinePhase
  questions: CheckpointQuestion[]
  answers: Record<string, string>
  resolved: boolean
  manual?: boolean  // true = artesanato
}
```

### 3.4 Fluxo Completo

```
START → IDEA
  ↓
coreops_next
  ↓
BRAINSTORM (executa BrainstormAgent)
  ↓
BrainstormAgent gera pergunta
  ↓
createCheckpoint(questions, manual=true)
  ↓
coreops_next retorna: { requires_input: true, manual: true }
  ↓
USUÁRIO responde com coreops_answer({ answers, confirm:true })
  ↓
Checkpoint resolvido → pode avançar para PLANNING
```

## 4. ArchitectAgent

### 4.1 Responsabilidade

Gera especificação de arquitetura antes da geração de tarefas:

```typescript
interface ArchitectureSpec {
  patterns: string[]           // "repository-pattern", "service-layer"
  folder_structure: string     // ASCII tree
  tech_decisions: {           // Decisões técnicas
    concern: string
    decision: string
    rationale: string
  }[]
  conventions: string[]       // Convenções de código
  key_abstractions: {         // Classes principais
    name: string
    responsibility: string
    file_path: string
  }[]
}
```

### 4.2 Integração com Brainstorm

Recebe `brainstorm_result` para informar decisões arquiteturais:

```typescript
await architect.execute({
  plan: executionPlan,
  brainstorm_result: brainstormResult,  // contexto do brainstorming
  workspace_path: '/path/to/workspace'
})
```

## 6. ErrorStore

### 6.1 Memória de Erros

Armazena erros e soluções para aprendizado automático:

```typescript
interface ErrorRecord {
  id: string
  error_signature: string   // Normalizada (sem line numbers/paths)
  root_cause: string
  fix_description: string
  project: string
  occurrence_count: number
  last_seen_at: number
}
```

### 6.2 Uso no Debugger

```typescript
// Antes de debugar
const priorMatches = errorStore.findSimilar(errorLog)
if (priorMatches.length > 0) {
  // Injeta solução anterior no prompt do debugger
  debuggerInput.prior_solution = priorMatches[0]
}

// Após debug bem-sucedido
errorStore.record(errorLog, debug.root_cause, debug.fix)
```

### 6.3 Normalização de Erros

```typescript
// Entrada: "Error at line 42: Cannot read property 'x' of undefined"
// Saída:  "error at line n: cannot read property 'x' of undefined"
```

Remove:
- Números de linha (`line 42` → `line n`)
- Paths de arquivo (`/src/app.ts` → `/FILE.ext`)
- UUIDs

## 8. API MCP

### 8.1 coreops_next

Retorna informações de bloqueio:

```json

  "requires_input": true,
  "checkpoint": { ... },
  "manual": true,
  "message": "⚠️ ATENÇÃO: Este checkpoint é MANUAL..."
```

## 10. Lições Aprendidas

### 10.1 Integração MCP

**Problema:** OpenCode/LLM respondia automaticamente checkpoints antes do usuário validar.

**Solução:** Campo `manual` + `confirm` explícito no `coreops_answer`.

### 10.2 TypeScript Strict

**Problema:** Novos campos em interfaces causavam erros em testes existentes.

**Solução:** Adicionar campos opcionais (`?`) ou atualizar factories de teste.

### 10.3 Stateful Agents

**Problema:** BrainstormAgent precisa manter estado entre perguntas.

**Solução:** Armazenar estado em `brainstorm_session` no ProjectState.

### 10.4 Modularidade

**Decisão:** Manter agentes separados (brainstorm, architect) em vez de unificar — facilita manutenção e testes.
